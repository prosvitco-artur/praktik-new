# Альтернативні способи публічного доступу до Lando сайту

Окрім Cloudflare Tunnel, є кілька способів надати публічний доступ до вашого Lando сайту на сервері.

## 1. Nginx як Reverse Proxy (Рекомендовано для продакшену)

**Переваги:**
- Стабільне рішення для продакшену
- Можливість налаштувати SSL сертифікат
- Швидкий та надійний
- Повний контроль над конфігурацією

**Недоліки:**
- Потрібен домен або статична IP
- Потрібно налаштувати DNS

**Встановлення:**
```bash
cd deploy
sudo ./setup-nginx-proxy.sh
```

Після встановлення сайт буде доступний за вашим доменом або IP адресою.

**Налаштування SSL (Let's Encrypt):**
```bash
sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d ваш-домен.com
```

## 2. Ngrok

**Переваги:**
- Швидке налаштування
- Автоматичний HTTPS
- Можливість використання власного домену (платно)

**Недоліки:**
- Безкоштовна версія має обмеження
- Тимчасові URL (безкоштовна версія)
- Потрібна реєстрація

**Встановлення та використання:**
```bash
# Встановлення
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null
echo "deb https://ngrok-agent.s3.amazonaws.com buster main" | sudo tee /etc/apt/sources.list.d/ngrok.list
sudo apt update && sudo apt install ngrok

# Реєстрація (отримайте токен на https://dashboard.ngrok.com/get-started/your-authtoken)
ngrok config add-authtoken ВАШ_ТОКЕН

# Запуск tunnel
ngrok http https://praktik-new.lndo.site
```

---

## 3. LocalTunnel

**Переваги:**
- Безкоштовний
- Не потрібна реєстрація
- Простий у використанні

**Недоліки:**
- Менш стабільний ніж ngrok
- Тимчасові URL
- Можливі обмеження швидкості

**Встановлення та використання:**
```bash
# Встановлення через npm
sudo npm install -g localtunnel

# Запуск tunnel
lt --port 443 --host https://praktik-new.lndo.site
# АБО якщо Lando працює на HTTP
lt --port 80
```

---

## 4. SSH Port Forwarding

**Переваги:**
- Не потрібно встановлювати додаткове ПЗ
- Безпечне з'єднання
- Працює через існуючий SSH доступ

**Недоліки:**
- Потрібен SSH доступ до сервера
- Не зручно для постійного доступу
- Потрібно тримати SSH сесію відкритою

**Використання:**
```bash
# На локальному комп'ютері
ssh -L 8080:localhost:443 користувач@сервер

# Потім відкрийте в браузері: http://localhost:8080
```

---

## 5. Tailscale / ZeroTier (VPN)

**Переваги:**
- Безпечний доступ через VPN
- Можливість налаштувати приватну мережу
- Безкоштовні версії доступні

**Недоліки:**
- Потрібно встановити клієнт на всіх пристроях
- Не публічний доступ (тільки для вашої мережі)

**Встановлення Tailscale:**
```bash
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up
```

Після цього ви зможете отримати доступ до сайту через приватну IP адресу Tailscale.

---

## 6. Прямий доступ через IP та порт

**Переваги:**
- Найпростіший спосіб
- Не потрібно додаткового ПЗ

**Недоліки:**
- Потрібно відкрити порт у firewall
- Небезпечно без SSL
- Потрібна статична IP адреса

**Налаштування:**
```bash
# Відкрити порт у firewall
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Перевірити IP адресу сервера
ip addr show

# Сайт буде доступний за адресою: http://IP_АДРЕСА:80
```

**Увага:** Цей метод небезпечний без SSL сертифікату!

## Рекомендації

1. **Для продакшену:** Використовуйте Nginx як reverse proxy з SSL сертифікатом
2. **Для швидкого тестування:** Cloudflare Tunnel або Ngrok
3. **Для приватного доступу:** Tailscale або ZeroTier
4. **Для тимчасового доступу:** SSH port forwarding

---

## Важливі зауваження

- Переконайтеся, що Lando запущено перед налаштуванням будь-якого методу: `lando start`
- Для продакшену завжди використовуйте HTTPS (SSL сертифікат)
- Налаштуйте firewall для захисту сервера
- Регулярно оновлюйте ПЗ для безпеки
