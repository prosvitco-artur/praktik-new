# Аналіз проекту та SEO покращення

## Що я зрозумів про проект

### Загальна інформація
- **Тип проекту:** WordPress тема для агентства нерухомості "Praktik"
- **Технологічний стек:** Sage 10 (Roots/Sage), Laravel Blade, Tailwind CSS, Carbon Fields
- **Призначення:** Платформа для публікації та пошуку об'єктів нерухомості

### Ключові функції
1. **7 кастомних post types:**
   - Кімнати (Rooms)
   - Квартири (Apartments)
   - Будинки (Houses)
   - Ділянки (Plots)
   - Гаражі (Garages)
   - Комерційна нерухомість (Commercial)
   - Дачі (Dachas)

2. **Система управління:**
   - Carbon Fields з 8+ вкладками кастомних полів
   - Детальна інформація про об'єкти (ціна, площа, локація, зручності)
   - Інтеграція з AirWP Sync для імпорту з Airtable

3. **Функціональність для користувачів:**
   - Потужна система фільтрів (ціна, площа, кімнати, тип)
   - Система обраного (favorites) для авторизованих та гостей
   - Галереї зображень з Fancybox
   - Адаптивний mobile-first дизайн

4. **Додаткові функції:**
   - Система відгуків
   - Контакти ріелторів
   - Шорткоди для форм пошуку

## Поточний стан SEO

### ✅ Що вже є:
- ✅ Підтримка `title-tag` (WordPress автоматично генерує заголовки)
- ✅ HTML5 семантична розмітка
- ✅ Responsive дизайн (mobile-first)
- ✅ Чисті URL (rewrite rules)
- ✅ Підтримка featured images
- ✅ Структурована навігація

### ❌ Що відсутнє (критичні SEO покращення):

#### 1. **Open Graph та Twitter Cards**
- Відсутні meta теги для соціальних мереж
- Немає preview зображень при шарингу
- Немає структурованих даних для соцмереж

#### 2. **JSON-LD структуровані дані (Schema.org)**
- Відсутні структуровані дані для:
  - RealEstateAgent (Агентство нерухомості)
  - Product/Offer (Об'єкти нерухомості)
  - BreadcrumbList (Навігаційні хлібні крихти)
  - Organization (Організація)
  - LocalBusiness (Локальний бізнес)

#### 3. **Breadcrumbs (Хлібні крихти)**
- Відсутня навігація breadcrumbs
- Важливо для UX та SEO

#### 4. **Meta Description**
- Немає кастомних meta descriptions
- WordPress генерує автоматично, але не оптимізовано

#### 5. **Canonical URLs**
- Відсутні canonical теги
- Може призвести до дублювання контенту

#### 6. **Alt тексти для зображень**
- Потрібно перевірити наявність alt атрибутів
- Важливо для accessibility та SEO

#### 7. **Robots Meta Tags**
- Немає контролю над індексацією окремих сторінок
- Немає noindex для служебних сторінок

#### 8. **Sitemap**
- Не видно XML sitemap
- Важливо для пошукових систем

#### 9. **Hreflang теги**
- Якщо планується багатомовність

#### 10. **Оптимізація зображень**
- Lazy loading
- WebP формат
- Оптимізація розмірів

## Рекомендовані SEO покращення

### Пріоритет 1 (Критичні)

#### 1.1. Open Graph та Twitter Cards
```php
// Додати в head секцію
<meta property="og:title" content="...">
<meta property="og:description" content="...">
<meta property="og:image" content="...">
<meta property="og:url" content="...">
<meta property="og:type" content="website">
<meta name="twitter:card" content="summary_large_image">
```

#### 1.2. JSON-LD структуровані дані
- RealEstateAgent schema для головної
- Product/Offer schema для об'єктів нерухомості
- BreadcrumbList schema
- Organization schema

#### 1.3. Breadcrumbs
- Навігаційні хлібні крихти на всіх сторінках
- JSON-LD BreadcrumbList

#### 1.4. Meta Description
- Кастомні meta descriptions для об'єктів
- Автоматична генерація з контенту

### Пріоритет 2 (Важливі)

#### 2.1. Canonical URLs
- Додати canonical теги на всі сторінки
- Особливо важливо для архівів з фільтрами

#### 2.2. Alt тексти
- Перевірити та додати alt для всіх зображень
- Автоматична генерація з назви об'єкта

#### 2.3. Robots Meta
- Контроль індексації
- noindex для служебних сторінок (favorites, search)

#### 2.4. XML Sitemap
- Генерація sitemap для об'єктів нерухомості
- Включення всіх post types

### Пріоритет 3 (Бажані)

#### 3.1. Оптимізація зображень
- Lazy loading
- WebP формат
- Responsive images (srcset)

#### 3.2. Page Speed
- Мінімізація CSS/JS
- Кешування
- CDN для статики

#### 3.3. Structured Data для відгуків
- Review schema для відгуків клієнтів
