# Функціональність поля Enable

## Опис

Ця функціональність додає поле `Enable` до всіх кастомних типів постів (room, apartment, house, plot, garage, commercial, dacha). Коли поле встановлено як `false`, пост автоматично видаляється.

## Функції

### 1. Автоматичне видалення постів
- Коли поле `Enable` встановлено як `false`, пост автоматично видаляється при збереженні
- Показується попередження користувачу про видалення поста

### 2. Приховування відключених постів
- Відключені пости не відображаються на фронтенді
- Вони автоматично виключаються з усіх запитів

### 3. Адмін інтерфейс
- Додана колонка `Enable` в списку постів
- Можливість швидко перемикати статус кліком на іконку
- Попередження при знятті галочки з поля

### 4. Безпека
- Перевірка прав доступу
- AJAX запити захищені nonce
- Запобігання нескінченним циклам

## Використання

### В адмінці:
1. При створенні/редагуванні поста ви побачите поле `Enable`
2. За замовчуванням воно увімкнено (`true`)
3. Якщо зняти галочку та зберегти, пост буде видалено
4. В списку постів можна швидко перемикати статус кліком на ✅/❌

### На фронтенді:
- Відключені пости автоматично не відображаються
- Вони не з'являються в пошуку, архівах або інших списках

## Технічні деталі

### Файли:
- `inc/carbonfields/post-meta.php` - додано поле Enable
- `inc/post-auto-delete.php` - основна логіка
- `languages/praktik-uk.po` - переклади

### Хуки:
- `save_post` - автоматичне видалення
- `pre_get_posts` - приховування з фронтенду
- `manage_posts_columns` - додавання колонки
- `wp_ajax_toggle_enable_status` - AJAX обробка

### Мета-поле:
- Назва: `crb_enable`
- Тип: checkbox
- Значення за замовчуванням: `true`

## Безпека

- Перевірка прав користувача
- Захист AJAX запитів nonce
- Валідація даних
- Запобігання XSS атакам

## Підтримка

При виникненні проблем:
1. Перевірте права доступу користувача
2. Переконайтеся, що Carbon Fields правильно встановлено
3. Перевірте консоль браузера на наявність JavaScript помилок
